Weekly Contribution: Please provide a summary of your contributions to the project during this week. Be specific about tasks, responsibilities, and any challenges you encountered. Describe how your work benefited the overall team and project.

Implemented browser-based document administration and multi-format ingestion for FP6. The application can now upload, list, replace, and delete UTF-8 TXT, text-based PDF, and DOCX documents. Uploaded documents are parsed on the server and sent through the existing chunking, MySQL, and ChromaDB pipeline, which makes their content available to the existing RAG question-answer workflow.

Added PHP upload validation for upload status, file size, extension, MIME type, and real HTTP upload state. Uploaded files receive random server-controlled storage names, while the original filename is stored separately as display metadata. Replacement is limited to browser-uploaded documents of the same format so the bundled Metro State source documents cannot be overwritten accidentally.

Added a normalized Python document loader using strict UTF-8 decoding for TXT, pypdf for text-based PDF files, and python-docx for Word paragraphs and tables. Invalid, encrypted, empty, binary, and non-extractable documents return actionable errors. Scanned PDFs are identified as requiring OCR, which is outside the current FP6 scope.

Expanded the database and ingestion helpers to preserve original filenames, parser status, errors, and chunk counts while replacing prior MySQL chunks and ChromaDB vectors without duplicates. Updated the browser interface with elapsed upload progress, document status, chunk counts, refresh controls, replacement/deletion controls, safe error messages, and live dashboard document/category counts. Deletion removes browser-managed uploads from file storage, MySQL, and ChromaDB while protecting the bundled Metro State documents.

Verified the milestone with 17 automated tests, real DOCX and text-based PDF extraction, and live HTTP upload, replacement, and deletion tests. A disposable upload changed the live indexed count from 28 to 29, and deletion returned it to 28 without removing the original demo document. The FP5 bundled-source regression remained stable at 27 documents, 77 MySQL chunks, and 77 ChromaDB vectors, and the existing Ask workflow continued retrieving the expected source.

The main challenge was coordinating secure browser uploads across PHP and Python while preserving one shared ingestion path for every supported format. This implementation prepares the project for FP7, which will create at least 25 manually reviewed evaluation questions and baseline evaluation metrics.
